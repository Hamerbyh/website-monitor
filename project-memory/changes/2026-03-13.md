## Deployment Notes
- For Dokploy, the app should connect to PostgreSQL through the Dokploy-internal host or service name rather than `127.0.0.1`.
- `RUN_DB_PUSH` is intended for controlled use during initial setup or schema changes, then should be turned back off.
- `BETTER_AUTH_URL` must match the final externally reachable site URL to avoid auth callback or cookie issues.
