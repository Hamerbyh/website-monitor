## Summary
- Added an optional Polymarket monitor that runs inside the existing Dokploy cron endpoint without adding new database tables.
- Added Telegram Bot API alert delivery for Polymarket price movement notifications.
- Added environment-driven Polymarket target configuration using market slug, market id, or event slug.
- Compared current Polymarket outcome prices with CLOB price history over a configurable lookback window.
- Documented the no-table tradeoff: repeated alerts can happen while the configured historical window remains above threshold.
- Split Polymarket checks into a dedicated cron endpoint so it can run on a separate 5-minute schedule from site due checks.
- Added compact Telegram lines for multi-market events, including option label, Yes/No prices, 24h volume, and liquidity.
- Added optional `marketLabels` filtering so multi-market event monitoring can target specific child markets such as one company.
- Added target-level `notifyAlways` support for snapshot notifications even when price movement is below threshold.

## Implementation Notes
- `POLYMARKET_MONITOR_TARGETS` is JSON and should use generic public slugs or ids in tracked examples.
- This implementation intentionally avoids storing market state, private Telegram credentials, or target-specific operational details in the repository.
- `/api/monitor/polymarket/check` uses the same `MONITOR_CRON_SECRET` auth pattern as the main due-check endpoint.
- Future table-backed work can add dedupe, per-target cooldowns, alert history, and UI-managed targets.
