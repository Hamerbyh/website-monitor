# Project Memory

This directory stores lightweight project memory for future development turns.

## Structure
- `changes/`: chronological change notes
- `index.md`: topic-based lookup for important changes

## Rule
- After each meaningful code change, append or add a dated note under `changes/`.
- Update `index.md` when a note introduces a new area or materially changes an existing one.
- Keep entries concise and focused on:
  - what changed
  - why it changed
  - follow-up or known issues
- Do not include sensitive values or user-provided private project details.
- Use generic placeholders for domains, emails, and environment-specific data.
